How it works.

A single-page website with an admin dashboard, backed by a database, with optional payments and email notifications. Everything auto-deploys from GitHub.

The Stack

LayerServicePurposeCost
HostingNetlifyWebsite + serverless APIFree
CodeGitHubVersion control + auto-deployFree
DatabaseSupabasePostgreSQL — stores everythingFree
PaymentsStripeCheckout + spot holds1.75% + 30c
EmailResendConfirmations + notificationsFree (100/day)

Monthly Cost

$0 for most clients. Stripe only charges per transaction. You only pay if you scale past the free tiers (which handles thousands of bookings).

Accounts needed.

Create these once. Each client gets their own Supabase project but shares your Netlify and GitHub accounts.

Required

GitHub

  • Sign up at github.com
  • Create an organization per client (e.g., vfit-studio)
  • Keeps client projects separate from your personal repos
Required

Netlify

  • Sign up at netlify.com (use GitHub login)
  • One account manages all client sites
  • Install the CLI: npm install -g netlify-cli
Required

Supabase

  • Sign up at supabase.com (use GitHub login)
  • Create one project per client
  • Free tier: 2 projects, 500MB, 50K rows
Optional

Stripe (for payments)

  • Sign up at stripe.com
  • Ideally the CLIENT creates this (their bank account for payouts)
  • Use test mode first, switch to live when ready
  • Select "Non-recurring payments" during setup
Optional

Resend (for emails)

  • Sign up at resend.com
  • One account for all clients
  • Free: 100 emails/day, $20/mo for 50K

Setup flow.

⏱ Total: ~30 minutes
Step 1

Create the repo

⏱ 2 min

Copy the template project folder. Create a GitHub org and repo for the client.

gh api user/orgs -X POST -f login=clientname-studio gh repo create clientname-studio/website --public --source=. --push
Step 2

Deploy to Netlify

⏱ 2 min

Link the repo and deploy. Every future push auto-deploys.

netlify link --name clientname netlify deploy --prod

Or: Netlify dashboard → Add new site → Import from GitHub → Select repo

Step 3

Create Supabase project

⏱ 5 min
  • Go to Supabase dashboard
  • New Project → name it after the client → region: Sydney
  • Wait ~30 seconds for it to spin up
  • Go to SQL Editor → paste the schema (see Database section)
  • Go to Settings → API → copy Project URL and service_role key
Step 4

Set environment variables

⏱ 2 min
netlify env:set SUPABASE_URL "https://xxxxx.supabase.co" netlify env:set SUPABASE_SERVICE_KEY "eyJhbGci..." netlify env:set ADMIN_KEY "choose-a-password" netlify env:set SITE_URL "https://clientname.netlify.app"
Step 5

Test everything

⏱ 5 min
  • Open the site → check it loads
  • Go to /admin.html → log in with your ADMIN_KEY
  • Create an event → check it appears on the website
  • Submit a booking → check it appears in admin
  • Submit a membership enquiry → check it appears in Requests
Step 6

Customise for the client

⏱ Varies

See the Customisation section below.

Database schema.

Paste this into Supabase SQL Editor and click Run. Creates all 8 tables.

create table if not exists events ( id uuid default gen_random_uuid() primary key, name text not null, type text not null, tickets_open timestamptz not null, session_date timestamptz not null, spots_total integer not null default 20, price_cents integer not null default 0, status text not null default 'active', created_at timestamptz default now() ); create table if not exists bookings ( id uuid default gen_random_uuid() primary key, event_id uuid references events(id), name text not null, email text not null, phone text, status text not null default 'held', stripe_session_id text, stripe_payment_id text, amount_cents integer default 0, held_at timestamptz default now(), confirmed_at timestamptz, created_at timestamptz default now() ); create table if not exists memberships ( id uuid default gen_random_uuid() primary key, name text not null, email text not null, phone text, plan text not null, sessions text, days text, times text, notes text, status text not null default 'new', created_at timestamptz default now() ); create table if not exists notifications ( id uuid default gen_random_uuid() primary key, email text not null, name text, interest text, type text default 'notify', notified boolean default false, created_at timestamptz default now() ); create table if not exists contacts ( id uuid default gen_random_uuid() primary key, name text not null, email text not null, phone text, message text, created_at timestamptz default now() ); create table if not exists schedule_slots ( id uuid default gen_random_uuid() primary key, day_of_week integer not null, time text not null, max_capacity integer not null default 4, status text not null default 'active', created_at timestamptz default now() ); create table if not exists members ( id uuid default gen_random_uuid() primary key, membership_id uuid references memberships(id), name text not null, email text not null, phone text, plan text not null, sessions_per_week integer not null default 1, status text not null default 'active', start_date date, notes text, created_at timestamptz default now() ); create table if not exists member_slots ( id uuid default gen_random_uuid() primary key, member_id uuid references members(id) on delete cascade, slot_id uuid references schedule_slots(id) on delete cascade, status text not null default 'active', created_at timestamptz default now(), unique(member_id, slot_id) );

Stripe setup.

⏱ 10 min
Step 1

Create Stripe account

Go to stripe.com. Select "Non-recurring payments". Stay in test mode initially.

Step 2

Get API key

Developers → API Keys → Copy the Secret key (starts with sk_test_...)

netlify env:set STRIPE_SECRET_KEY "sk_test_..."
Step 3

Add webhook

Developers → Webhooks → Add endpoint

  • URL: https://yoursite.netlify.app/.netlify/functions/stripe-webhook
  • Events: checkout.session.completed and checkout.session.expired

Copy the Signing secret (starts with whsec_...)

netlify env:set STRIPE_WEBHOOK_SECRET "whsec_..."
Step 4

Go live

When ready: toggle Stripe to live mode, replace sk_test_ with sk_live_, create a new live webhook and update the secret.

Email notifications.

⏱ 5 min
Step 1

Create Resend account

Go to resend.com → sign up → go to API Keys → Create.

netlify env:set RESEND_API_KEY "re_..." netlify env:set OWNER_EMAIL "client@email.com"

Custom domain.

⏱ 5 min + DNS propagation
Step 1

Buy a domain

Client buys from any registrar (Namecheap, GoDaddy, Crazy Domains, etc.)

Step 2

Add to Netlify

Netlify → Site → Domain management → Add custom domain → enter the domain

Step 3

Update DNS

Point the domain's DNS to Netlify (they'll show you the exact records). SSL auto-provisions.

Step 4

Update SITE_URL

netlify env:set SITE_URL "https://clientdomain.com.au"
