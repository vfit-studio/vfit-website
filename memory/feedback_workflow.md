Build everything fully before showing the client — don't iterate with the client in the loop until it's polished.

**Why:** Fred wants to demonstrate the full power of what can be built. Half-finished work undermines the pitch.

**How to apply:** When building for a new client, complete all features (website + admin + payments + emails) before presenting. Use version branches (A, B, C, D) for design alternatives so he can compare and pick.

Additional feedback from this session:
- Emoji icons look amateur on luxury sites — use typography or thin-line elements instead
- "Memberships" as a name doesn't land for boutique PT — use language like "Studio Sessions"
- Countdown widgets need crystal clear labels — session date, bookings open date, and what the countdown is counting to must all be obvious
- Home page should tease, not tell everything — simple cards with "View Details" that go to sub-pages
- Videos as background bleeds on sub-pages, not on the home page panels
- The hero image should be the client's actual brand image, not stock
- Nav should be clean — one style, not mixed text + block buttons
- Admin needs to be organised by the business (Run Club / Pi'lattes / Members) not by data type (Events / Bookings)
- Everything must work on iPhone — most users will view on mobile
- Don't use random stock videos — they need to match the brand quality
